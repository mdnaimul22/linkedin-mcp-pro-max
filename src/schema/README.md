# `src/schema/` Directory Guide

## এই ফোল্ডারটি কী কাজ করে?
এই ফোল্ডারের কাজ হলো **Security Guard** বা ফিল্টার হিসেবে কাজ করা। প্রজেক্টের যেকোনো ডেটার স্ট্রাকচার (Type) এবং বৈধতা (Validation) এখানে চেক করা হয়। 

আমরা এখানে **Pydantic** মোডেল ব্যবহার করি, যা নিশ্চিত করে যে আমাদের কোডে কোনো গারবেজ (Garbage) বা ভুল ডেটা ইনজেক্ট হচ্ছে না।

## কখন আমরা Schema ব্যবহার করবো?
১. **যখন ডেটা অবজেক্ট অনেক বড় এবং কমপ্লেক্স হয়:**
   যেমন `Profile` এর ভেতরে `Education`, `Experience` এবং `Skills` এর মতো নেস্টেড লিস্ট থাকতে পারে। ব্রাউজার বা API থেকে আসা কাঁচা ডেটা (Raw dictionary) সরাসরি ব্যবহার না করে, আমরা তাকে Schema এর ছাঁচে ফেলে পিউরিফাই (Purify) করি।

২. **টাইপ সেফটি (Type Safety) নিশ্চিত করতে:**
   কোড লেখার সময় যাতে IDE (VS Code/Cursor) অটো-সাজেশন দেয় এবং ডেটা টাইপ নিয়ে কোনো এরর না আসে। (যেমন: `start_date` স্ট্রিং না হয়ে একটি `Date` অবজেক্ট হবে)।

৩. **টুলের আউটপুট ফরম্যাট হিসেবে:**
   MCP টুলসগুলো যখন LLM বা ইউজারকে ডেটা রিটার্ন করে, তখন এই Pydantic স্কিমাগুলো থেকে একটি পরিচ্ছন্ন JSON তৈরি করে দেওয়া হয় (`model.model_dump()`)।

---
**মনে রাখবেন:** Schema ফাইলগুলো (যেমন `profile.py`, `jobs.py`) কখনোই ডাটাবেসে ডেটা সেভ করে না বা ব্রাউজার ওপেন করে না। এরা শুধুমাত্র ডেটার "আকার" বা "ডিজাইন" ডিফাইন করে!
